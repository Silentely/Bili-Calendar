使用说明

  • Apple 日历：打开设置 > 密码与账户 > 添加账户 > 其他 > 添加已订阅日历
  • Google 日历：在左侧"我的日历"下点击"添加其他日历" > "从URL添加"
  • Outlook：在日历视图中点击"添加日历" > "从Internet"

功能特色

  • 智能识别连载状态：连载中番剧自动设置每周重复，完结番剧仅保留首播时间
  • 生成的日历可添加到 Apple 日历、Google 日历、Outlook 等
  • 每集更新时间会自动添加到您的日历中
  • 日历链接长期有效，无需重复订阅
  • 支持 PWA 安装，可添加到手机桌面像原生应用一样使用
  • 支持暗黑模式，夜间追番更舒适
  • 支持外部 ICS 日历聚合，最多合并 5 个日程源
  • 自动缓存历史记录，快速切换多个 UID
  • 支持中英文界面，自动检测语言偏好
重要提醒：您的追番数据必须设置为公开，否则无法获取

使用教程

  • 公开追番列表 — 打开B站 App → 设置 → 安全隐私 → 开启「公开我的追番列表」
  • 获取 UID — 访问B站个人空间，URL 中的数字即为您的 UID
  • 生成订阅链接 — 在上方输入 UID，点击「生成订阅」按钮
  • 添加到日历 — 复制链接并添加到 Apple 日历、Google 日历或 Outlook

常见问题

追番列表不公开怎么办？

请在B站 App 中进入 设置 → 安全隐私 → 开启「公开我的追番列表」。网页端可在个人空间的「账号设置」中修改。不公开的列表无法被第三方工具读取。

更新频率是多少？

日历中的番剧更新时间基于B站官方数据，连载中番剧会自动设置每周重复规则。实际更新时间以B站为准。

支持哪些日历应用？

支持所有兼容 iCal/ICS 格式的日历应用，包括 Apple 日历、Google 日历、Outlook、Thunderbird 等。

如何自部署？

项目支持 Docker 一键部署。运行 docker run -p 3000:3000 ghcr.io/silentely/bili-calendar 即可启动。详见 GitHub 仓库。

数据是否安全？

服务端不存储任何用户数据。您的 UID 仅用于实时查询B站 API，不会被记录或分享。项目完全开源，可自行审计。
